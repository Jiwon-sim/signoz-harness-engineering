# /ecc — 7단계 컴플라이언스 절차

대상 작업: **$ARGUMENTS**

판정 기준·복잡도별 생략 규칙은 `.claude/steering/ecc.md` 를 따른다. 이 명령은 그 절차를 **실행**한다.
강제(하드 차단)가 아니라 **보고상 게이트**로 운영한다 — 검증은 가능하면 실제 수행, 실패는 정직하게 보고.

## 절차

복잡도를 먼저 판단한다(오타·1줄 → 1·3만 / 일반 → 전 단계 / API·DB·보안 → 전 단계 + 사람 승인).
각 단계는 체크박스로 결과를 보고한다.

- [ ] **1. 계획 수립** — 영향 범위·접근 방식 명시.
- [ ] **2. 영향 분석** — 의존성·호출처·에디션 판별. 필요 시 `/analyze-feature $ARGUMENTS`.
- [ ] **3. 구현** — 변경 위치에 따라 위임:
  - Go(`pkg/`·`ee/`) → `go-implementer` 에이전트
  - Frontend(`frontend/`) → `react-implementer` 에이전트
- [ ] **4. 빌드 검증** — 가능하면 `make go-build-community` / `pnpm build` 실제 실행.
- [ ] **5. 보안 검토** — `code-reviewer` 에이전트(보안 전담).
- [ ] **6. 코드 리뷰** — `/code-review`(빌트인, 정확성·기술 품질) → 사람 승인. ※ `code-reviewer` 에이전트로 중복하지 않는다.
- [ ] **7. 데드코드 정리** — 미사용 import/export/변수·디버그 로그 제거.

## 미통과 처리
- 문제와 원인을 정직하게 보고한다.
- 필요하면 사람의 판단을 받아 다음 단계 진행 여부를 결정한다.
- **하드 차단하지 않는다.** "실패 시 다음 단계 진입 금지" 식 게이트를 만들지 않는다.

## 출력
각 단계 체크 결과 + 마지막에 종합 판정(통과 / 보류 사유). 검증을 실제로 돌리지 않았으면 그렇다고 명시한다.
